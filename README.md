# Разнесение Росстата на сетку 1×1 км

Проектный практикум (501), МФТИ, 2026. Кейс 1 от АНО «ИКИИГУ»: разнести
агрегированные региональные показатели Росстата по сетке 1×1 км без простого
усреднения, опираясь на внешние пространственные признаки.

## Команда
- Фёдор Кравченко - метод затухания.
- Полина Кузнецова - регрессионная модель + WorldPop, координация.
- Е. Неяскина - модель максимальной энтропии (MaxEnt).

## Что сделано

Пилот - три региона разного типа:
- Московская область (агломерация),
- Краснодарский край (агропромышленный регион),
- центральная часть Якутии: бокс 400×400 км вокруг Якутска.

Для всей Якутии 1×1 км не помещается в память, поэтому отдельно построена
сетка 10×10 км на весь регион.

Показатели Росстата:
- `Y477090007` - отгрузка товаров собственного производства;
- `Y477120029` - продукция сельского хозяйства.

Подходы к разнесению (от простого к сложному):
1. **Baseline.** Равномерно по площади ячеек.
2. **По расстояниям до городов (v1).** Чем ближе к городу, тем выше вес.
3. **WorldPop (v2).** Отгрузка распределяется пропорционально населению в ячейке.
4. **Регрессия + WorldPop (v3).** На 85 регионах учим
   `log(отгрузка) ~ log(население) + log(гор.население) + log(инвестиции)`.
   Главный коэффициент - 2.77 при городском населении. На сетке используем
   как эластичность: `вес = население^2.77`.

## Регрессия, подробности

- Обучали на 85 регионах (Росстат, 2023).
- R² in-sample: 0.84.
- 5-fold кросс-валидация: R² = 0.78 ± 0.05. Переобучения нет.
- 5 регионов с большими остатками: Ингушетия и Дагестан (другая структура
  экономики, модель завышает), Амурская обл., Приморский край и ЕАО
  (Дальний Восток, логистика плохо ловится через население).
- Финальный v3 пока сделан только для отгрузки. С/х закрыт методом v1
  (расстояния), регрессию для с/х оставили на потом.

## Данные

Все ноутбуки обращаются к данным по путям относительно корня репозитория
(`./data/raw/...`), поэтому Jupyter надо запускать из корня репо.

**Parquet с Росстатом.** Приходит от заказчика. Кладём в:
```
./data/raw/Массив Данных в формате Parquet/Показатели по регионам.parquet
```

**WorldPop.** Скачать растр для России за 2020 год с
https://www.worldpop.org/ (Population Counts, Russia, 2020, Constrained, 100m)
и положить в `./data/raw/worldpop/russia_pop_2020.tif`. Для каждого пилота
растр потом обрезается по границе региона - это делается прямо в
`02_grid.ipynb`, обрезанные куски сохраняются в `./data/processed/pop_*_z.tif`.

**Границы регионов.** Подтягиваются автоматически через `osmnx` и кешируются
в `./data/processed/border_*.gpkg`.

## Как запустить

```bash
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

Положить parquet и WorldPop в `data/raw/` (см. выше), потом из корня репо:

```bash
jupyter notebook
```

И прогнать ноутбуки в порядке `01 -> 03 -> 02`:
1. `notebooks/01_data.ipynb` - проверяем parquet, находим нужные indicator_code,
   достаём региональные значения для пилотов.
2. `notebooks/03_worldpop.ipynb` - проверка растра WorldPop.
3. `notebooks/02_grid.ipynb` - главный ноутбук. Тут всё остальное:
   сетки 1×1 км, baseline, методы v1/v2/v3, Джини, top-10%, карты.

В `02_grid.ipynb` есть промежуточные тестовые ячейки. Финальный метод v3
живёт в блоке с функцией `method_v3`, и там же кросс-валидация регрессии.

## Метрики качества

- **Джини** по значениям ячеек - насколько неравномерно распределилось.
- **Топ-10%** - какая доля показателя осела в самых нагруженных 10% ячеек.
- **R²** регрессии на отложенной выборке (5-fold CV) - для метода v3.

## Что не работает или работает не очень

- На Московской области v3 даёт Джини 0.99 и топ-10% = 99%. С одной стороны
  это согласуется с реальным доминированием Москвы, с другой - эластичность
  2.77 фактически сжимает всю отгрузку в несколько ячеек, а периферия
  получает почти ноль. Это не баг, но и не «победа», - скорее ограничение
  выбранной формы веса.
- Модель училась на показателях обрабатывающей промышленности и плохо
  переносится на регионы с другой структурой экономики (Северный Кавказ,
  Дальний Восток).
- Якутия покрыта на 1×1 км только в центральном боксе 400×400 км. Для всего
  региона - 10×10 км, иначе не хватает памяти.
- Признаков мало: только население, гор. население и инвестиции. Хочется
  ещё VIIRS, OSM-инфраструктуру, ESA WorldCover, ВРП по отраслям.
